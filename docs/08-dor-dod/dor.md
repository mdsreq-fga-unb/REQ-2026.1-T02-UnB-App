# 8.1 Definition of Ready (DoR)

> Acordo entre o time e o Product Owner que indica quando um requisito está preparado para entrar em uma Sprint.

## Checklist do DoR

| # | Critério | Descrição |
|---|----------|-----------|
| 1 | **Informação necessária** | O requisito tem detalhes suficientes para o time entender o que deve ser feito, sem ambiguidades. |
| 2 | **Cabe em uma Sprint** | O requisito é pequeno o bastante para ser concluído em um único ciclo. |
| 3 | **User Story** | O requisito está descrito no formato de história de usuário. |
| 4 | **Critérios de aceite + BDD** | Critérios de aceitação e especificações BDD estão presentes. |
| 5 | **Interface mapeada** | Se envolve interface, ela está definida e descrita. |

!!! example "Formato de User Story"
    **Como** [tipo de usuário], **quero** [ação], **para** [benefício/objetivo].
