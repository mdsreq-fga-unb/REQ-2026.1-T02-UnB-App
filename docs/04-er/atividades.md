# 4.1 Atividades e Técnicas de ER

> Para cada uma das 6 atividades da Engenharia de Requisitos, informe as técnicas que serão utilizadas e como serão aplicadas no contexto do projeto.

---

=== "Elicitação e Descoberta"
    | Técnica | Como será utilizada no projeto |
    |---------|-------------------------------|
    | **Questionários** | Aplicação de formulários digitais com o público 60+ para identificar as principais problemas no uso do SIGAA. |
    | **Entrevistas** | Realização de reuniões via Microsoft Teams com a representante Maria Fátima para aprofundar o entendimento dos gargalos operacionais. |

=== "Análise e Consenso"
    | Técnica | Como será utilizada no projeto |
    |---------|-------------------------------|
    | **Priorização MoSCoW** | Classificação das funcionalidades (Must have, Should have, etc.) para definir o que compõe o MVP focado em autonomia. |
    | **Reuniões de Sprint** | 2 reuniões por semana com a equipe para alinhar a viabilidade técnica das necessidades elicitadas. |

=== "Declaração de Requisitos"
    | Técnica | Como será utilizada no projeto |
    |---------|-------------------------------|
    | **User Stories** | Escrita de requisitos sob a perspectiva do usuário idoso (Ex: "Como aluno 60+, quero ver minha carteirinha offline"). |
    | **Especificação Prosa** | Descrição textual detalhada das regras de negócio, como o funcionamento do cache para acesso offline. |

=== "Representação de Requisitos"
    | Técnica | Como será utilizada no projeto |
    |---------|-------------------------------|
    | **Prototipagem de Alta Fidelidade** | Criação de telas no Figma focadas em alto contraste e tipografia ampliada para validação visual. |
    | **Rich Picture** | Representação visual do fluxo de interação atual vs. a solução proposta para alinhar o contexto do negócio. |

=== "Verificação e Validação"
    | Técnica | Como será utilizada no projeto |
    |---------|-------------------------------|
    | **Testes de Usabilidade** | Observação direta (via compartilhamento de tela) de usuários reais realizando tarefas no protótipo ou MVP. |
    | **Sprint Review** | Demonstração quinzenal do incremento de software para a cliente validar se atende aos critérios de acessibilidade. |

=== "Organização e Atualização"
    | Técnica | Como será utilizada no projeto |
    |---------|-------------------------------|
    | **GitHub Projects (Kanban)** | Centralização e rastreamento do status de cada requisito (Épicos e User Stories) durante o ciclo de vida. |
    | **Controle de Versão (Git)** | Manutenção do histórico de alterações nos documentos de visão e requisitos conforme o feedback do cliente. |
